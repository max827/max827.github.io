如何使用hexo+GitHub来搭建一个个人博客

​ 很多人说自己搭建完博客之后，第一篇博客也就出来了……所以理所应当的，我的第一篇博客就是怎么来搭建一个博客啦。虽然网上已经有很多的详细教程了，但是由于每个人的操作系统和环境配置不同，不太可能找到完全适合自己的一篇教程，该遇到的坑还是会遇到。我就是看了一个教程视频才萌发了搭建博客的想法，这个教程20分钟就完成了搭建，而我实际上却整整用了一天的时间才算是搞定了hexo，中间遇到了很多莫名其妙的问题，幸亏相关的教程在网上是真的多，还是可以找到相应的解决方法的。以下将是我搭建博客的全过程，希望能给遇到相同问题的人提供一点帮助。

  • 操作系统：win10
  • 环境配置：无，一片空白

环境准备

可以参照hexo官网的教程：hexo官网

里面有讲到在安装hexo之前，必须先要安装Node.js和Git这两个东西：

Node.js安装

Node.js可以去官网上下载：Node.js官网

下载好之后直接默认安装即可

Git安装

Git由于不能翻墙，去官网上下载会很慢，所以可以直接从GitHub上下载：国内Git下载地址

如果Node.js安装好了，可以在win10的命令提示符中查看他们的版本（npm是跟随Node.js一起安装的），如果可以正常显示，说明你安装成功了

接下去的操作基本上都将使用命令行操作，你可以使用win10自带的命令提示符，也可以使用刚刚安装的Git Bash来完成命令行的操作，这里我也不知道什么原因，有些命令在win10的cmd上可以运行，到了Git Bash上又不行，有些又只能在Git Bash运行，所以下面的命令可能在这两者之间来回切换，我下面的命令中，前面带着美元符号“$”的，就是在Git Bash上输入的命令；没有带的，就是在win10的cmd上输入的命令，如果你自己试验时发现输入的命令在win10的cmd上不行，就换成Git Bash，两种选择供你使用，下图是如何打开Git Bash（点击鼠标右键出现）：

点击Git Bash Here,打开界面如下：
